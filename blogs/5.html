Developer Diary

February 11th, 2022

Dynamic Library

It's crucial for our game to be able to load a library at run time, so it would be more lightweight and generic. Essentially, in our case, the game development team would require an API, in this case the function prototypes, from the chat team and then load that in anywhere in the game program that requires chat functions. Loading libraries in dynamically would allow the application to not have to load everything at the startup of the app, which would significantly worsen the performance of our game.

Application Programming Interface (API)

An API would have to be provided to the game dev team so that they could implement the chat features. I suppose what the chat team will be doing in the coming weeks is to work on documenting the API so that it would be easy to implement.
